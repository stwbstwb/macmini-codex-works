# SEO記事生成仕様

## 目的

柏谷横浜社労士事務所のコンテンツマーケティングとして、Google検索から評価される人事労務SEO記事を継続的に作成する。

記事は「人事労務だよりの紹介」ではなく、人事労務だよりを素材・きっかけとして、中小企業の経営者・人事労務担当者に役立つ専門的な記事へ深掘りする。

## 記事の立ち位置

- 社労士という人事労務の専門家が、客観的な立場から情報提供する。
- 事務所の宣伝、CTA、相談誘導は入れない。
- 本文内に `人事労務だより`、`掲載されていた`、`出典PDF` などの出典管理語を出さない。
- 出典PDFの号・掲載箇所・元トピックは、通知メール、ログ、確認用管理情報に記録する。

## テーマ選定

優先するテーマ:

- 法改正、制度改正、施行日・適用日が重要なテーマ
- 厚生労働省、行政、公的機関の全国的な動き
- 中小企業の実務に影響する労務管理、助成金、社会保険、給与計算、就業規則関連
- 検索需要があり、既存記事とカニバリしにくいテーマ
- 最新号に良いテーマがない場合の、過去号からの有用テーマ

原則除外するテーマ:

- `ニュース` 区分の単発ニュース
- 山口県、福島県など、特定地域に限定された制度・事案
- 都道府県名、地域労働局、労基署名が中心となる地域限定ニュース
- 送検事案など、個別事件性が強くSEO記事化しにくい素材
- 業務スケジュール、法人税、消費税、戸籍など、人事労務SEOの主軸から外れる素材
- 既存記事とのカニバリが強いテーマ

最新の未作成号で除外後に3件分の有効テーマがない場合は、その号を作成済みにせず、同じ実行内で次の未作成号へ遡る。遡っても3件分の有効テーマが見つからない場合は、件数を埋めるために無理に記事化せず `blocked_insufficient_articles` として停止し、通知で要確認にする。

## 記事構成ルール

手動の記事構成プロンプトを正とし、次を必ず満たす。

- H2見出しは7個以上
- 最初のH2は `はじめに`
- 最後のH2は `まとめ`
- H1は本文内に使わない
- H2/H3/H4をテーマごとに設計する
- `事例`、`成功例` を見出しに入れない
- 検索意図と再検索意図を満たす網羅性を持たせる
- 3記事生成時は、構成が似通わないようにする
- 記事タイトルや構成を機械的な型に固定しない
- `中小企業` は読者設定として扱い、毎回タイトルに入れる必須語にしない
- `導入前チェックリスト`、`申請前のチェックリスト`、`実務チェックリスト` などの汎用H2を定型的に使わない
- チェック項目的な内容は、記事テーマに合うH2へ言い換える
- 3件中2件以上が `中小企業が確認したい`、`中小企業が見直したい`、`中小企業が押さえたい` などの定型タイトルに寄る場合は品質NGにする
- 3件中2件以上で、まとめ直前H2がチェックリスト型になる場合は品質NGにする

## 本文生成ルール

手動の記事本文プロンプトを正とし、次を必ず満たす。

- 記事構成の全H2/H3/H4に本文を書く
- 各見出しブロックは200文字以上
- 記事全体は3000文字以上
- 詳しくない読者でも理解できるように、具体的な実務対応まで書く
- 他サイトのコピーや類似表現にならないよう、オリジナル本文にする
- `そして`、`また` などの接続詞は多用しない
- 生成AIらしい定型的な冒頭・まとめを避ける
- 3記事の `はじめに`、`まとめ`、H2構成の類似度をチェックする

## 事実確認

- 法律、制度、日付、数値は一次情報を確認対象にする。
- 主な確認先は、厚生労働省、e-Gov法令検索、日本年金機構、国税庁、労働局、自治体など。
- 未確認の法律・日付・数値が残る場合は、公開可能扱いにしない。
- 下書き保存は可能だが、確認ログに未確認項目を残す。

## 自動実行

- 実行頻度: 毎月1日 0:00
- 生成件数: 3件
- WordPressステータス: 下書き
- WordPress日付: 実行日 9:00 を3件すべてに設定
- 投稿者: `ksrfp`
- タグ: 設定しない
- スラッグ: 設定しない
- カテゴリ: 記事内容から自動判定
- Arkhe CSS Editor: 指定CSSを反映
- アイキャッチ: 記事ごとに内容に合う写真品質の背景画像を毎回新規生成し、タイトルを中央配置する
  - 過去生成画像、既存の同名画像、同テーマの別記事画像、同一バッチ内の他記事画像の再利用は禁止
  - 記事ブリーフ作成後に更新された写真ソースだけを新規生成扱いにする
  - 画像生成プロンプトは記事タイトル、元トピック、PDF内区分、ラベル、素材抜粋から構図を分岐する
  - 助成金・採用は採用面談、給与計算は給与/社会保険実務デスク、無期転換は契約更新面談、安全衛生は工場/倉庫等の現場確認、年金・ライフプランは相談/ビジネス街など、テーマごとに背景を変える
- 確認用テキスト: 指定Google Driveフォルダに保存する。ファイル名先頭の `YYMMDD` は確認用テキストの作成日を使う
- Google Driveに同名の確認用テキストが既にある場合は新規作成ではなく更新する。同名候補が複数ある場合は更新日時が新しい候補を更新し、候補数をログに残す
- 通知メール: 成功・失敗問わず送信し、3件それぞれの号・掲載箇所・元トピックを記載する
- 通知メール: SMTP/sendmailを最大3回再試行する。成功マーカーは実送信後だけ作成する
- WordPress保存前に3件すべてのペイロード、本文品質、ファクト確認、画像品質、選定ポリシーを一括検査する
- 1件でもNGの場合、WordPressへは1件も作成しない
- WordPress同名タイトルは、下書き・予約・保留・非公開・公開済みを全件確認する。状態履歴にない同名下書き、別トピックの同名下書き、同名公開済み投稿がある場合は停止する
- WordPress保存、WordPress検証、Drive保存、最終通知はステップ別に最大3回再試行する。ただしmanifest不一致、件数不一致、同一run状態不一致など決定的な仕様違反は再試行で成功扱いにしない
- WordPress保存ログ、WordPress検証ログ、Drive保存メタデータ、最終通知payload、通知送信済みマーカーは、現在の `post_payloads_latest.json` のSHA-256と一致する場合だけ現在runの成功根拠にする
- WordPress投稿作成/更新に成功したら、Arkhe CSS保存など後続工程の前に投稿ID、トピックキー、画像SHA-256を状態履歴へ記録する。後続工程で失敗しても、次回は同じトピックの既存下書きを復旧対象として扱う
- 同じペイロード・同じアイキャッチ画像SHA-256で既存下書きを再試行する場合のみ、既存メディアIDを再利用できる。これは同一runの冪等化であり、新規記事や画像未生成時の代替手段として過去画像を使うことは不可
- 下書き済み・投稿済みのテーマは選定しない。3件に満たない場合は履歴ありテーマで無理に埋めず、未作成号を新しい順に遡る
- `partially_drafted` または `draft_saved_for_review` の号は、生成段階の履歴更新で `analyzed` に戻さない
- 前回までのプロセス中断などで `partially_drafted` または必要件数未満のWordPress下書きIDが号単位履歴に残っている場合、次回月次実行ではいきなり新規3件生成へ進まず、まず現在manifestで再開できるか判定する。部分下書きPDF名と現在manifestのPDF名が一致し、既に作成済みのトピックキーを現在manifestが含む場合だけ再開を許可する。再開できる場合はWordPress保存以降または画像生成引き継ぎを続行し、再開不能な場合だけ `blocked_partial_draft_issue` として通知して同じ号の下書き過剰作成を防ぐ

## 品質ゲート

以下を満たさない記事は、WordPress下書き保存前に要確認とする。

- H2 7個以上
- `はじめに` 開始
- `まとめ` 終了
- 各見出しブロック200文字以上
- 3000文字以上
- 禁止表現なし
- 事務所CTAなし
- 出典管理語なし
- 実務ポイントあり
- 一次情報確認計画あり
- 3記事間の類似チェックOK
- タイトル重複なし
- タイトル型の偏りなし
- 構成型の偏りなし
- 全記事のまとめ直前がチェックリスト型になっていない
- 3件中2件以上が定型タイトル・チェックリスト型構成に寄っていない
- アイキャッチ背景が記事内容に合った写真品質である
- アイキャッチ背景が毎記事ごとに新規生成されている
- 同一バッチ内で写真ソースが重複していない
- コード生成の簡易イラスト背景を本番下書きに使っていない
- アイキャッチ画像は通常ファイルかつ非空ファイルである
