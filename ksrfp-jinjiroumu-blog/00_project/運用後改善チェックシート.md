# ksrfp-jinjiroumu-blog 運用後改善チェックシート

## 位置づけ

このチェックシートは、開発完了後・定期運用開始後に確認する改善項目を管理する。

開発段階で実施可否を判断できないため、`進捗管理チェックシート.md` から分離する。

## 開始条件

- WordPress下書き保存までの月次自動実行が安定している
- 実際の記事公開後、GSC・GAで評価できるだけのデータが蓄積している
- 投稿後の検索流入、クリック率、閲覧状況を比較できる期間がある

## 1. 投稿後データ確認

- [ ] 投稿後のGSCデータを定期確認する
- [ ] 投稿後のGAデータを定期確認する

## 2. 成果記録

- [ ] 検索流入が伸びたテーマを記録する
- [ ] クリック率が低い記事のタイトル改善候補を出す

## 3. リライト・ネタ元拡張

- [ ] 古くなった記事のリライト候補を出す
- [ ] 人事労務だより以外のネタ元を追加するか検討する

## メモ

- 初回公開後すぐには判断せず、一定期間のデータを見て評価する。
- 法改正・助成金・制度情報の記事は、検索成果とは別に情報鮮度の観点でもリライト対象にする。
