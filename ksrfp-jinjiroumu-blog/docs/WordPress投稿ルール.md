# WordPress投稿ルール

## 投稿カテゴリ

記事テーマと本文内容から、以下のカテゴリのうち最も適切なものを1つ自動設定する。

| カテゴリ | スラッグ | ID |
| --- | --- | --- |
| ライフプランニング | `life-planning` | 13 |
| 助成金 | `subsidy` | 5 |
| 労務管理 | `labor-management` | 7 |
| 年金 | `pension` | 15 |
| 給与計算 | `payroll` | 25 |
| 給付金 | `benefits` | 10 |

## 自動判定の基本方針

- 助成金・奨励金・補助金が主題の場合は `助成金` を優先する。
- 給付金・教育訓練給付が主題の場合は `給付金` を優先する。
- 年金制度、遺族年金、在職老齢年金が主題の場合は `年金` を優先する。
- 給与計算、割増賃金、社会保険料徴収、算定基礎、現物給与が主題の場合は `給与計算` を優先する。
- 扶養、健康保険と年金、生活設計寄りのテーマは `ライフプランニング` を候補にする。
- 労働時間、36協定、就業規則、有給休暇、休憩、ハラスメント、安全衛生、雇用管理は `労務管理` を候補にする。
- 明確な一致がない場合は `労務管理` を既定カテゴリにする。

## 下書き保存と日付設定

- 投稿曜日: 月曜日
- 投稿時刻: 朝9:00
- タイムゾーン: `Asia/Tokyo`
- 投稿ステータス: `draft`
- 予約投稿として公開待ちにはせず、WordPressには下書きで保存する。
- 日付は従来どおり月曜日朝9:00を設定する。
- 直近未来の月曜日9:00に既存予約投稿がある場合は、翌週月曜日9:00の日付を設定する。
- 翌週にも予約投稿がある場合は、さらに翌週へ送る。
- 外部API有効化後は、WordPress上の `future` 投稿も確認して日付枠の重複を避ける。

## Arkhe CSS Editor

投稿画面の「Arkhe CSS Editor」には以下を入力する。

```css
.c-postContent h2{
border-bottom: solid 1px black;
padding: 0.3em 0;
}
.c-postContent h3{
background: #efefef;
padding: 0.5em 0.6em;
}
.c-postContent h4{
padding: 0.25em 0.5em;
border-left: solid 3px #265d9b;
}
```

確認メモ:

- 投稿編集画面上の入力フィールド名は `arkhe_css_editor_meta`。
- WordPress REST APIレスポンスにはこのmetaは公開されない。
- XML-RPCは `https://ksrfp.com/ksrfp/xmlrpc.php` が空応答のため利用不可。
- 2026-06-19の初回投稿テストでは、管理画面で `arkhe_css_editor_meta` に入力し、プレビュー表示でCSS反映を確認した。
- 2026-06-20のREST API下書き保存では、Arkhe側のREST設定 `arkhe_css_front` / `arkhe_css_editor` に指定CSSが入っていることを確認した。投稿メタがRESTに公開されないため、API実行時は既存CSSを保持したままこのREST設定へ指定CSSを追記・確認する。
