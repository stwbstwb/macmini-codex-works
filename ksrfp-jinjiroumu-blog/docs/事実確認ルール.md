# 事実確認ルール

## 基本方針

法律、制度、日付、数値データを記事内に入れる場合は、必ず根拠を確認する。

## 必ず確認するもの

- 法律名
- 制度名
- 施行日
- 適用日
- 申請期限
- 対象企業・対象労働者
- 金額、人数、割合、件数、時間数
- 調査名、調査対象、調査時期
- 助成金の支給額、要件、受付状況

## 優先する情報源

1. 厚生労働省、労働局、年金機構、自治体などの一次情報
2. e-Gov法令検索
3. 官公庁のパンフレット・PDF
4. 出典PDFである人事労務だより

## 公開判定

- 法律、制度、日付、数値データの確認項目が1件でも `unverified` の場合、公開可能とは扱わない。
- 下書き記事の品質がOKでも、根拠確認が残っている場合は `publication_ready` を `false` にする。
- WordPress API連携では、`safe_to_publish` が `true` のときだけ公開・下書き投稿の次工程へ進める。
- 出典PDFの記載を本文で使う場合も、制度・日付・数値が現行情報として通用するかを公的情報で確認する。
- 確認済みにする場合は、確認日、確認URL、根拠メモを保存する。

## 禁止事項

- 根拠が確認できない法律・数字を断定しない。
- 古い年度の助成金情報を最新情報として書かない。
- 地域限定制度を全国制度のように書かない。
- PDFに載っているだけで、現在も有効と決めつけない。
- 出典が曖昧な調査データを一般論として使わない。

## 実装上の扱い

- `03_generated/articles/fact_check_items_latest.md` に、法律・制度・日付・数値の確認項目を出力する。
- 各確認項目には、公的情報へ当たるための候補URLを出力する。
- 未確認項目が残る場合、`publication_gate` は `blocked_until_verified` とする。
- `article_quality_check_latest.md` では、下書き品質と公開可能判定を分ける。
- WordPress公開投稿へ進む前に、未確認項目を解消する。
