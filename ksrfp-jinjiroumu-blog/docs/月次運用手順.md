# 月次運用手順

この手順は、柏谷横浜社労士事務所ブログ向けの記事生成を毎月運用するときの確認用です。

## 基本方針

- 月次実行は毎月1日0:00にCodexオートメーション `ksrfp-weekly-run` が開始する。
- 1回の実行で3記事を作成する。
- WordPressには公開ではなく下書きとして保存する。
- Google Driveには確認用テキストを保存する。
- 画像生成は追加課金を要する手段を使わず、ChatGPT/Codex側の利用中プラン内で使える画像生成ツールに引き継ぐ。
- 成功・要確認の結果は `stonewebstoneweb@gmail.com` へ通知する。

## 通常の確認順

1. 通知メールを確認する。
   件名が `[ksrfp-jinjiroumu-blog] 自動実行成功: ok` なら、WordPress下書き、Drive確認用テキスト、最終契約テストまで完了している。

2. WordPress下書きを確認する。
   通知本文または `07_logs/weekly_latest.json` にある投稿IDを開き、アイキャッチ、タイトル、見出し、本文、カテゴリ、日付を確認する。

3. Google Drive確認用テキストを確認する。
   通知本文または `03_generated/review-texts/review_text_item_*.json` にあるDrive URLを開き、レビュー用テキストが3件揃っていることを確認する。

4. 問題なければ、進捗管理チェックシートへ人間レビューOKを記録する。
   記録先は `00_project/進捗管理チェックシート.md`。

## 画像生成待ちになった場合

ステータスが `needs_image_generation_tool` の場合は、Python側では画像を生成しない。

1. `07_logs/weekly_latest.json` の `image_generation_required_items` を確認する。
2. 各記事の `featured_image_prompt` に従い、Codex画像生成ツールで写真背景を作成する。
3. 生成画像を各 `featured_image_source_path` に保存する。
4. 次のコマンドを実行する。

```bash
/Users/ug/.cache/codex-runtimes/codex-primary-runtime/dependencies/python/bin/python3 06_automation/continue_after_external_image_sources.py
```

この続行スクリプトは、画像再構築、WordPress下書き保存、WordPress検証、Drive保存、最終通知まで進める。

## Drive保存待ちになった場合

ステータスが `needs_drive_upload_plugin` の場合は、WordPress下書きと検証は完了しているが、確認用テキストのDrive保存だけが残っている。

1. `03_generated/review-texts/` の日付付き `.txt` 3件を確認する。
2. Google Driveプラグインで、指定フォルダ `1vFh7U9F2DhItpYRzVvJ2qRKUpvmOD7jp` へ3件を保存する。
3. 返ってきたDrive URLを使い、次の形式で記録する。

```bash
/Users/ug/.cache/codex-runtimes/codex-primary-runtime/dependencies/python/bin/python3 06_automation/record_drive_plugin_uploads.py \
  --drive-upload 'ファイル名1.txt=https://drive.google.com/file/d/...' \
  --drive-upload 'ファイル名2.txt=https://drive.google.com/file/d/...' \
  --drive-upload 'ファイル名3.txt=https://drive.google.com/file/d/...'
```

4. 最終通知を送る。

```bash
/Users/ug/.cache/codex-runtimes/codex-primary-runtime/dependencies/python/bin/python3 06_automation/send_manual_full_test_notification.py
```

## 成功判定

最終的に次を満たしていれば成功。

- `07_logs/weekly_latest.json` の `status` が `ok`
- `07_logs/final_run_contract_latest.json` の `status` が `ok`
- `07_logs/final_run_contract_latest.json` の `failed_count` が `0`
- `07_logs/notifications/latest_notification.json` の `status` が `sent`
- WordPress下書き3件を人間が確認済み
- Drive確認用テキスト3件を人間が確認済み

## 異常時

- `blocked_*`、`partial`、`error` は成功扱いにしない。
- WordPressに3件すべての下書きが揃っていない場合は、手動で追加投稿せずログを確認する。
- 同じPDF号や同じテーマを再生成しそうな場合は、`08_state/processed_pdfs.json`、`08_state/topic_history.json`、`08_state/scheduled_posts.json` を確認する。
- 状態が分からない場合は、`07_logs/weekly_latest.json` と `07_logs/final_run_contract_latest.json` を最優先で見る。

## 保存

人間レビューまで完了したら、次を行う。

1. 成功状態のバックアップを作成する。
2. 進捗管理チェックシートを更新する。
3. コードや仕様に変更があればGitへコミットする。
