# テーマ選定ロジック

## 入力

- `02_analysis/topic-selection/newsletter_topic_candidates.csv`
- `02_analysis/seo/gsc_top_queries.csv`
- `02_analysis/cannibalization/posted_articles_inventory.csv`

## 評価項目

| 項目 | 内容 |
| --- | --- |
| PDF基礎スコア | PDF内の法改正・ニュース・助成金・労務管理キーワードから算出 |
| SEO需要 | GSCクエリとの一致、表示回数、クリック数から算出 |
| 実務影響 | 中小企業の人事労務対応に関係するキーワードで加点 |
| 鮮度 | PDF年月をもとに新しい号を加点 |
| カニバリリスク | 投稿済み記事タイトルとの類似度が高い場合に減点 |
| 履歴 | 下書き保存済み・投稿済みのテーマは再選定しないよう減点 |

## 最新号と過去号の扱い

- 最新号にカニバリリスクが低く、未使用の候補がある場合は最新号を優先する。
- 最新号に記事化できる候補がない場合は、過去号の未使用候補から月単位で決まるシードを使って候補を選ぶ。
- 未確認ファクトで止まったテーマは履歴に残すが、投稿済み扱いにはしない。
- WordPressに下書き保存できたテーマは、次回以降の重複回避対象にする。

## 出力

- `02_analysis/topic-selection/topic_selection_scores.csv`
- `02_analysis/topic-selection/topic_selection_report.md`

## 注意点

- スコアはテーマ選定の一次判定であり、最終記事化前には一次情報確認を行う。
- GSC需要は既存サイトで確認できる関連需要であり、外部検索ボリュームそのものではない。
- 類似度はタイトル中心の簡易判定であり、本文レベルのカニバリ確認は後続工程で強化する。
- 過去号フォールバックはランダム性を持たせるが、同じ月内では再現性のある選定になる。
