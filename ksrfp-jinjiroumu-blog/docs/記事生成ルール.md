# 記事生成ルール

## 目的

人事労務だよりから選定したテーマをもとに、中小企業の経営者・人事労務担当者に役立つSEO記事を作成する。

## 構成ルール

- H1は記事タイトルとして扱い、本文中には使わない。
- H2は7個以上を基本とする。
- 最初のH2は「はじめに」とする。
- 最後のH2は「まとめ」とする。
- H3は必要な論点を分解して入れる。
- 記事構成には「事例」「成功例」を入れない。
- Markdown形式で出力する。
- 記事タイトル、H2、H3は、記事テーマ・検索意図・読者の悩みに合わせて変える。
- 「中小企業」は想定読者として意識するが、全記事タイトルへ機械的に入れない。
- 「導入前チェックリスト」「申請前のチェックリスト」「実務チェックリスト」などの汎用H2を定型的に使わない。
- チェック項目的な内容が必要な場合も、テーマに合う見出しに言い換える。
- まとめ直前のH2を毎回チェックリスト型にしない。

## 本文ルール

- 各H2/H3に本文を入れ、省略しない。
- 全体で3000文字以上を目安にする。
- 中小企業の実務に落とし込む。
- 専門用語は、詳しくない読者にも伝わるように説明する。
- 「再検索する必要がなくなる」「検索上位10記事」という表現は使わない。
- 接続詞の多用を避け、自然な文章にする。
- 事務所アピールや行動喚起は入れず、社労士という人事労務の専門家が客観的に情報提供する立ち位置で書く。

## 事実確認ルール

- 法改正、制度情報、助成金、行政資料は一次情報を確認する。
- 施行日、適用日、申請期限、対象企業、対象労働者を確認する。
- 出典PDFの情報が最新でない可能性がある場合は、本文で断定しない。
- 地域限定の制度は、全国向けの一般記事として書かない。
- 法律名、制度名、日付、数値データは `fact_check_items_latest.md` に抽出し、未確認のまま公開しない。
- 根拠が見つからない情報は、本文から削除するか「出典PDFでは紹介されている」範囲の表現にとどめる。

## 現在の実装

- 推奨テーマに応じて記事作成ブリーフを生成する。
- 一次情報確認計画を生成する。
- 記事構成案を生成する。
- Markdown本文ドラフトを生成する。
- 文字数、H2/H3、客観トーン、事務所アピール・CTA禁止表現などの機械チェックを行う。
- 法律・日付・数値のファクトチェック項目を抽出する。
- ファクトチェック確認済みレジストリに登録された根拠を反映する。
- WordPress投稿用HTMLとJSONペイロードを生成する。
- 未確認ファクトやWordPress接続テスト未完了が残る場合は、ペイロードを `ready_to_send=false` にする。
- アイキャッチ画像の生成プロンプト、ファイル名、altを生成する。
- アイキャッチ画像は背景写真に記事タイトルを合成し、青帯・白太字テキストを上下左右中央に配置する。
- アイキャッチ背景は写真品質を必須とする。コードで描画した簡易イラスト背景は緊急確認用に限定し、WordPress下書き保存へ進めない。
- 写真背景が未生成の場合は `requires_photorealistic_source` として止め、AI画像生成で記事内容に合う写真背景を作成してからタイトル合成・投稿へ進む。
- 写真品質ゲートは画像計画、WordPressペイロード、WordPress書き込み直前の3か所で判定する。
- 3記事バッチでは、タイトル重複、全タイトルへの「中小企業」定型挿入、H2構成の過度な類似、全記事のまとめ直前チェックリスト化を品質ゲートで検出する。
